## 1. 项目背景与目标

### 1.1 为什么需要因果效应估计？

在已完成的因果发现工作中，我们使用PC算法发现了变量之间的**因果关系结构**（因果图），但存在以下关键局限：

| 已完成工作（因果发现） | 待完成工作（因果效应估计） |
|----------------------|-------------------------|
| 发现 "威灵仙 → 颈痛" 有因果关系 | 威灵仙能让颈痛改善多少？ |
| 知道因果关系的方向（谁影响谁） | 是正向影响（恶化）还是负向影响（改善）？ |
| 构建了完整的因果网络图 | 每条边的权重（效应大小）是多少？ |

**核心问题**：PC算法只回答"**有没有**因果关系"，不回答"**有多强**因果关系"。

### 1.2 本计划的目标

在已有因果图的基础上，对发现的**药物→症状**因果关系进行**定量效应估计**：

1. **估计因果效应大小**：使用药物后，症状改善/恶化了多少？
2. **确定效应方向**：是正向效应（症状加重）还是负向效应（症状改善）？
3. **评估效应显著性**：这个效应在统计上是否显著？
4. **生成带权重的因果图**：边上的数字表示效应大小

## 2. 理论基础

### 2.1 因果发现 vs 因果推断

```
因果发现 (已完成)               因果推断 (本计划)
─────────────────               ─────────────────
问题: 是否存在因果关系？        问题: 因果效应有多大？
方法: PC算法                    方法: 倾向性评分匹配 + ATE估计
输出: DAG结构（有向图）          输出: ATE值（效应大小）
用途: 理解变量间的关系           用途: 量化治疗效果，指导临床
```

### 2.2 核心概念：ATE (Average Treatment Effect)

**ATE** = 平均处理效应，表示"使用药物"与"不使用药物"在结果上的平均差异。

```
ATE = E[Y(1) - Y(0)]

其中:
- Y(1) = 使用药物后的症状得分（潜在结果）
- Y(0) = 不使用药物后的症状得分（潜在结果）
- E[·] = 期望（对所有患者求平均）
```

**问题**：我们无法同时观察到同一患者的 Y(1) 和 Y(0)（因果推断的基本问题）

**解决方案**：通过**倾向性评分匹配 (PSM)** 创建"伪对照组"

### 2.3 倾向性评分匹配 (PSM) 原理

```
目标: 找到"相似"的患者，一个用药，一个不用药

步骤:
1. 估计倾向性评分 e(x) = P(T=1 | X=x)
   - T = 是否使用目标药物 (0/1)
   - X = 混杂因素（年龄、性别、基线症状等）

2. 匹配:
   处理组 (用药) ←→ 对照组 (不用药)
   匹配标准: 倾向性评分相近

3. 计算ATE:
   ATE = 平均(处理组症状变化 - 对照组症状变化)
```

## 4. 分析流程

### 步骤1: 定义研究问题

对于每一条药物→症状因果边，需要明确定义：

| 要素 | 说明 | 示例 |
|------|------|------|
| 处理变量 (Treatment) | 是否使用目标药物 | med_威灵仙_t (0/1) |
| 结果变量 (Outcome) | 后续症状严重程度 | 颈痛_t1 (0/1/2) |
| 基线症状 | 用于调整基线差异 | 颈痛_t |
| 混杂因素 | 同时影响用药决策和症状的因素 | 性别、年龄、基线诊断等 |

### 步骤2: 确定混杂因素

**混杂因素定义**：同时影响处理变量和结果变量的因素

**方法1：基于因果图**
```
在因果图中，混杂因素 = 同时指向 treatment 和 outcome 的变量的父节点
```

**方法2：领域知识**
```
常见混杂因素：
- 人口学特征：性别、年龄
- 基线症状：症状_t（严重程度）
- 基线诊断：诊断_t（影响用药决策）
- 合并用药：其他药物_t（多药联合治疗）
```

### 步骤3: 倾向性评分估计

**目标**：估计每个患者使用该药物的概率

**方法**：使用逻辑回归模型，以混杂因素为预测变量，以是否用药为目标变量进行训练，然后预测每个患者的用药概率。

**输出**：为每个时序对计算一个倾向性评分（0到1之间的数值）

### 步骤4: 匹配患者

**目标**：为每个用药患者找到倾向性评分相近的未用药患者作为对照

**方法**：
- 分离处理组（用药）和对照组（未用药）
- 设定匹配卡尺（通常为0.2倍倾向性评分的标准差）
- 使用1:1近邻匹配，为每个处理组患者匹配1-2个对照

**匹配后输出**：形成配对数据，每组包含一个用药患者和一个（或多个）特征相似的对照患者

### 步骤5: 平衡性检验

**目的**：验证匹配是否成功消除了混杂偏倚

**方法**：计算标准化差异 (Standardized Mean Difference, SMD)

**评估标准**：
- SMD < 0.1：平衡良好
- 0.1 ≤ SMD < 0.2：可接受
- SMD ≥ 0.2：平衡不佳，需重新匹配或调整方法

**输出**：生成平衡性评估报告，展示每个混杂因素在匹配前后的SMD变化

### 步骤6: 估计ATE

**目标**：在匹配数据的基础上，计算平均处理效应

**可选方法**：

| 方法 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| 简单均值差 | 直接比较处理组和对照组的症状变化 | 简单直观 | 未利用基线信息 |
| 回归调整 | 用回归模型调整基线差异 | 提高精度 | 依赖模型正确性 |
| 双稳健估计 | 结合倾向性评分和结果回归 | 两种模型只要一个正确即可 | 计算复杂 |

**建议**：优先使用双稳健估计方法，因为它对模型误设定具有鲁棒性

### 步骤7: 统计推断

**目标**：评估ATE的统计显著性

**方法**：
- 计算标准误差
- 构建95%置信区间
- 进行假设检验（t检验）

**结果解读示例**：
- ATE = -0.42：使用药物后症状平均改善0.42级
- 95% CI [-0.68, -0.16]：置信区间不包含0，说明效应显著
- p < 0.01：强烈拒绝"无效应"假设

### 5.3 实现流程概述

1. **加载因果发现结果**
   - 读取因果图结构（从 `causal_dag.pkl` 加载NetworkX Graph对象）

     e. 记录结果（ATE、SE、p值、样本量等）

4. **生成可视化**
   - 绘制带权重的因果图（边上标注效应值和方向）
   - 生成效应大小排序图
   - 绘制平衡性评估图

5. **生成报告**
   - 汇总所有因果效应估计结果
   - 生成详细的Markdown报告

## 10. 成功标准

1. **完整性**: 对因果发现识别的所有药物→症状边都估计了ATE
2. **质量性**: 匹配后的SMD < 0.1（平衡良好）
3. **可信性**: 大部分效应有合理的置信区间
4. **可解释性**: 报告清晰，临床医生能理解结果
5. **可复现性**: 代码和数据的完整记录

## 附录：数据不支持ATE估计时的退路策略

### A. 为什么需要退路策略？

ATE估计依赖于较强的假设，数据质量不满足时可能出现：

| 问题 | 表现 | 影响 |
|------|------|------|
| **样本量不足** | 某药物使用频率<5% | 匹配失败，置信区间过宽 |
| **阳性率极端** | 某药物90%以上或10%以下使用 | 倾向性评分聚集，无法区分 |
| **完全分离** | 某因素能完美预测用药 | 逻辑回归不收敛 |
| **重叠性差** | 处理组和对照组特征无交集 | 无法匹配 |

### B. 相关性 vs 因果效应：关键区别

**重要提醒**：相关性不等同于因果效应

```
场景示例：威灵仙 → 颈痛

实际情况：医生倾向于给"严重颈痛"患者开威灵仙

相关性分析结果：
"用威灵仙的人 → 颈痛更重" (正相关)
→ 错误结论：威灵仙无效或有害

因果效应（ATE）结果：
控制基线严重程度后
"用威灵仙的人 → 颈痛改善更明显" (负向因果效应)
→ 正确结论：威灵仙有效
```

| 维度 | 相关性 | ATE (因果效应) |
|------|--------|----------------|
| 问题 | "用这个药的人，症状是否更好？" | "这个药**导致**症状改善了吗？" |
| 假设 | 无 | 无混杂偏倚 |
| 问题 | 混杂偏倚（选择性偏倚） | 需满足可识别性条件 |
| 适用场景 | 探索性分析 | 因果结论 |

### C. 数据质量检查清单

在尝试ATE估计前，先进行以下检查：

#### 1. 样本量检查

| 处理组样本量 | 阳性率 | 推荐方法 | 报告方式 |
|--------------|--------|----------|----------|
| ≥30 | 10%-90% | PSM + ATE | 完整报告（ATE、CI、p值）|
| 10-29 | 10%-90% | PSM + ATE | 报告但标注"样本量小，谨慎解读"|
| <10 | 任意 | **改用相关性** | 明确说明"仅报告相关性，非因果效应"|
| 任意 | >90% 或 <10% | 改用IPW（逆概率加权） | 报告但标注"阳性率极端"|

#### 2. 重叠性检查

绘制倾向性评分分布图：

```
处理组分布: ████████████████
对照组分布:       ████████████
               ↑ 无重叠区域
```

**判断标准**：
- 处理组和对照组的倾向性评分分布有明显重叠：可继续匹配
- 严重分离或无重叠：放弃ATE，改用相关性分析

#### 3. 共同支撑假设检查

对每个个体计算倾向性评分后：
- 检查是否所有个体的倾向性评分都在合理范围内（如0.1-0.9）
- 如果某个群体的倾向性评分全部接近0或1：截断该群体或使用IPW

### D. 分层报告策略

根据数据质量，采用不同的报告格式：

| 层级 | 条件 | 报告内容 | 可信度 | 标签 |
|------|------|----------|--------|------|
| **A级** | 样本量≥30, 匹配率≥80% | ATE + 置信区间 + p值 | 高 | "因果证据" |
| **B级** | 样本量10-29, 匹配率50-80% | ATE + 警告标签 | 中 | "初步证据（谨慎解读）" |
| **C级** | 样本量<10 或 匹配率<50% | 仅报告相关性 | 低（明确标注） | "关联证据（非因果）" |

### E. 混合报告格式示例

对于不同数据质量的药物-症状对，采用不同格式：

#### E级：高质量因果证据

```markdown
### 威灵仙 → 颈痛

| 指标 | 值 | 解读 |
|------|-----|------|
| **处理组样本量** | 45 / 724 (6.2%) | - |
| **匹配成功率** | 82% | 可进行因果推断 |
| **ATE (因果效应)** | -0.42 [-0.68, -0.16], p<0.01 | **显著改善** |
| **Pearson相关** | -0.38, p<0.001 | 与ATE方向一致 |

**结论**：威�灵仙对颈痛有显著的因果改善作用（A级证据）
```

#### C级：仅能报告相关性

```markdown
### 浙贝母 → 耳鸣

| 指标 | 值 | 解读 |
|------|-----|------|
| **处理组样本量** | 8 / 724 (1.1%) | 样本量过小 |
| **匹配成功率** | 25% | **无法进行因果推断** |
| **ATE** | - | 无法估计 |
| **Pearson相关** | -0.21, p=0.08 | 弱负相关，不显著 |

**结论**：由于样本量过小，仅报告关联性，无法得出因果结论（C级证据）
```

#### 特殊情况：阳性率极端

```markdown
### 茯神 → 反酸

| 指标 | 值 | 解读 |
|------|-----|------|
| **处理组样本量** | 156 / 724 (21.5%) | - |
| **阳性率** | 78% | 倾向性评分聚集，匹配不稳定 |
| **ATE (IPW)** | -0.12 [-0.35, 0.11], p=0.31 | 因果效应不显著 |
| **ATE (回归调整)** | -0.08 [-0.28, 0.12], p=0.42 | 因果效应不显著 |
| **Pearson相关** | -0.15, p=0.02 | 弱负相关，显著 |

**结论**：因果效应不显著，相关性显著但可能存在混杂偏倚（B级证据）
```

### F. 相关性作为补充的规范

无论能否估计ATE，都应报告以下指标作为补充：

| 指标 | 说明 | 局限性 |
|------|------|--------|
| **原始相关** | Pearson相关系数 | 不控制混杂 |
| **条件相关** | 控制年龄、性别后的偏相关 | 仍有遗漏变量 |
| **分层相关** | 按基线症状严重程度分层的相关 | 样本量进一步分散 |

### G. 实现建议

在代码中实现自动判断逻辑：

```
FUNCTION estimate_effect(treatment, outcome, data):
    """根据数据质量自动选择合适的方法"""

    // 步骤1：检查样本量
    n_treated = 统计 data[treatment] 中值为1的数量
    n_total = 统计 data 的总行数
    positivity_rate = n_treated / n_total

    // 步骤2：判断使用哪种方法
    IF n_treated >= 30 AND 0.1 <= positivity_rate <= 0.9 THEN
        // 高质量数据：尝试PSM + ATE
        ate_result = CALL estimate_ate_with_psm(data, treatment, outcome)
        ate_result.evidence_level = 'A'
        ate_result.method = 'PSM + ATE'

    ELSE IF 10 <= n_treated < 30 AND 0.1 <= positivity_rate <= 0.9 THEN
        // 中等质量数据：尝试PSM + ATE，但标注警告
        ate_result = CALL estimate_ate_with_psm(data, treatment, outcome)
        ate_result.evidence_level = 'B'
        ate_result.method = 'PSM + ATE (谨慎解读)'

    ELSE
        // 低质量数据：仅报告相关性
        ate_result = CALL calculate_correlation(data, treatment, outcome)
        ate_result.evidence_level = 'C'
        ate_result.method = '相关性分析（非因果）'

    END IF

    RETURN ate_result
END FUNCTION
```

```

                    │ 开始：检查数据质量 │
                    └────────┬────────┘

                │                         │

    阳性率10-90%？  阳性率10-90%？  任意      任意

```

### I. 总结

1. **相关性不能替代因果效应**，但可以作为补充证据
2. 采用**分层策略**：根据数据质量选择合适的方法
3. **诚实报告**：明确区分"因果证据"、"初步证据"和"关联证据"
4. 不要强行估计ATE，如果数据不支持，就诚实报告相关性并标注局限性
5. 在报告中使用**证据等级标签**，帮助读者理解结果的可信度
